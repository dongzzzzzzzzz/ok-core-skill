# 输出正反例

这个文档用于约束 `property-advisor` 的最终回答风格。

## 反例：把内部过程直接展示给用户

下面这种输出是不合格的：

```markdown
第一步：硬约束过滤
搜索结果：15 条样本

过滤项 | 标准 | 结果
预算 | <= AUD 10,000/月 | 全部通过
...

第二步：样本内横向比较
最低：...
最高：...
中位数：...

第三步：候选评估
房源 | 价格 | 图片质量 | 分析
...
```

为什么不合格：

- 这是分析过程，不是用户要的最终建议
- 用户需要的是“看哪几套、为什么、风险是什么”
- 这种输出会让人感觉模型在展示自己的工作流，而不是帮用户做决定

```markdown

2. 2203/58 Clarke St
价格：A$489/周（约 A$2,117/月）
适合你的原因：Southbank 里性价比突出，预算压力最小
需要确认：到你工作地点的真实高峰通勤时间
链接：<url>

3. 81 A'Beckett St
价格：A$699/周（约 A$3,027/月）
适合你的原因：CBD 内更稳妥，适合更看重通勤确定性的情况
需要确认：楼层、朝向和噪音情况
链接：<url>

风险提醒：
- Paragon Apartments 的超低价更像短租或异常价格，不建议当作正常月租参考
- 这批结果缺少面积、卫浴和设施信息，目前更适合初筛，不适合直接定房

下一步：
- 先用 Google Maps 验证这 3 套到公司或学校的早高峰通勤
- 再联系房东确认面积、卫浴、家具和租约条款
```

为什么合格：

- 先给结论
- 只展示用户需要的部分
- 每条推荐都有理由，也有未确认风险
- 没有把中间分析过程暴露出来

```markdown

- 现有房源大多只提供标题、位置和价格

- 面积、卫浴、设施
- 真实通勤和周边便利度

- 放宽区域或卧室条件，先把样本量提高到 10 条以上
```
